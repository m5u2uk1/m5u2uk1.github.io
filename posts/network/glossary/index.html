2層アーキテクチャ

  • ネットワークをアクセス層とディストリビューション層の2つの層に分割する階層型ネットワーク設計です。

AAA

  • AAAは、ネットワークセキュリティやアクセス制御などの分野で使われる用語で、「Authentication（認証）」、「Authorization（認可）」、「Accounting（アカウンティングまたは会計）」の3つの機能を指します。

BPDUガード

  • BPDUガードは、スイッチポートで受信したBPDU（Bridge Protocol Data Unit）を検出し、そのポートをブロックする機能。誤ってネットワークのトポロジを変更することを防ぐ。

CAPWAP

CDP

  • CDP（Cisco Discovery Protocol）は、Cisco製のネットワークデバイスが同じネットワーク上の他のCiscoデバイスを自動的に検出し、その情報を交換するためのプロトコルです。CDPは、ネットワークトポロジのマッピングやデバイスの特性情報の取得などに使用され、Ciscoデバイス同士がお互いに通信する手段として機能します。

CDP のメリット

  • レイヤ 2 で動作するので、設定がおかしくて接続できなかったり IP を忘れてしまった隣接機器からも機器の情報をゲットできる

CISCO DNA CENTER

  • Ciscoが提供するネットワーク管理プラットフォームです。このプラットフォームは、統合されたソフトウェアアプリケーションスイートを提供し、エンタープライズネットワークの自動化、セグメンテーション、分析、セキュリティの強化などの機能を提供します。

DA

  • Designated Router 代表ルーター
  • OSPF（Open Shortest Path First）ルーティングプロトコルを使用するネットワークにおいて、マルチアクセスネットワーク（LANなど）上でルーティング情報の集約と拡散を行う特別なルータです。

DAI

  • DAI（Dynamic ARP Inspection）は、ARP スプーフィング攻撃からネットワークを保護するためのセキュリティ機能。ARP トラフィックを検査して信頼性のあるエントリのみを許可する。

DHCP コンフリクト

  • 別のクライアントに重複した IP アドレスが割り振られている状態。よろしくない。
    • 想定原因
      1. 重複した　DHCP サーバー
        • 異なる DHCP サーバーが同じネットワークに存在し、それらが異なる IP アドレスを同じクライアントに提供しようとする場合、コンフリクトが発生します。
      2. DHCP サーバーの構成エラー
        • DHCP サーバーが正しく構成されていない場合、重複した IP アドレスの割り当てが発生する可能性があります。
      3. ネットワークの変更
        • サブネット変更、DHCP サーバの移動が行われず、クライアントが異なるサブネットのIPを取得しようとする
      4. DHCP リリースの不備
        • DHCP RELEASE を正しく行わなかった場合、同じ IP アドレスが他のクライアントに再割り当てられ、競合が発生します。

DHCP スヌーピング

  • 不正な DHCP からの不正な DHCP 応答を防ぐしくみ。

DSSS

  • 直接シーケンススペクトラム拡散（DSSS）は、IEEE 802.11標準（Wi-Fi）で使用される一種の物理層技術。

DTP

  • Dynamic Trunking Protocol。DTPは、Ciscoスイッチ間でトランク接続を確立するためのプロトコルで、自動的にトランクを形成するか、またはそれを無効にする役割を果たします。

Dynamic Desirable

  • Cisco Discovery Protocol（CDP）やDynamic Trunking Protocol（DTP）など、動的なプロトコルを使用して、スイッチポートのトランキング（VLAN情報の伝送）を自動的に設定するモードを指します。

EIGRP

  • EIGRP（Enhanced Interior Gateway Routing Protocol）は、Ciscoが開発したプロプライエタリなルーティングプロトコルです。EIGRPは、距離ベクター型とリンクステート型の特徴を組み合わせた高度なハイブリッドプロトコルです。

EIGRP はデフォルトでルーティングテーブルのルートのメトリックをどう計算してる？

  • 帯域幅と遅延の値を見て計算しとる

K値

  • EIGRP の文脈で出る K 値は、経路メトリックの計算に使用される重みつけ係数を指す。

内部EIGRP

  • 単一のAS内で使用される。

FHRP

  • FHRP（First Hop Redundancy Protocol）は、ネットワークデバイスがデフォルトゲートウェイとして提供する冗長性を確保するためのプロトコルです。

FHRPなプロトコル群

  • HSRP (Ciscoプロプラ)
  • VRRP (IEEE標準)

Flexconnect

  • FlexConnectは、CiscoのワイヤレスLAN（WLAN）ソリューションで使用される機能の一つです。FlexConnectは、無線クライアントがローカルサイトに接続された際に、そのトラフィックを中央のWLANコントローラではなく、ローカルのアクセスポイント（AP）で処理するためのモードです。これにより、トラフィックが中央のコントローラを経由せずにローカルで処理され、帯域幅の効率化や低遅延の利点があります。

GRE

  • GRE（Generic Routing Encapsulation）は、異なるネットワークプロトコルをトンネリングするためのプロトコル。

GRE over IPsec

  • GRE over IPsecは、Generic Routing Encapsulation（GRE）トンネリングをセキュリティで保護するためにIPsecを使用する仕組み。

HSRP

  • HSRP（Hot Standby Router Protocol）は、冗長性を提供するためのCiscoのプロトコルの1つです。主にデフォルトゲートウェイの冗長化に使用されます。
  • HSRPは、複数のルーターが同じ仮想IPアドレスを共有し、アクティブなルーターとスタンバイルーターの役割を持ちます。アクティブなルーターはトラフィックを処理し、スタンバイルーターはアクティブなルーターが利用できなくなったときに、アクティブなルーターの役割を引き継ぎます。

HSRPのプリエンプト機能

  • プライオリティ値が高いデバイスが必ずアクティブルータとなるようにできる機能。デフォルトの場合はプライオリティが高くても既にアクティブルータが存在する場合はなれんのよ

IP SLA

  • IP SLA（IP Service Level Agreement）は、Ciscoのネットワークデバイスで提供される機能で、ネットワークの性能を監視し、トラブルシューティングやサービス品質の向上に役立つツール

ISATAP

  • ISATAP（Intra-Site Automatic Tunnel Addressing Protocol）は、IPv6ネットワークをIPv4インフラストラクチャ上に導入するためのトンネリング技術の一つです。ISATAPは、IPv6ホストがIPv6パケットをIPv4ネットワーク上で転送するための方法を提供します。

ISR

  • Cisco IOS ISR（Integrated Services Router）は、Ciscoのルーター製品ラインの一部で、複数のネットワークサービスを単一のプラットフォームで提供するために設計された統合型のルーターです。ISRは、データ、音声、ビデオ、セキュリティ、ワイヤレス、クワリティ・オブ・サービス（QoS）などの機能を単一のデバイスでサポートし、柔軟性と効率性を提供します。

LACP

  • LACP（Link Aggregation Control Protocol）は、ネットワークリンクのアグリゲーション（バンドル）を制御するための通信プロトコルです。LACPは、IEEE 802.1AX規格に基づいており、複数の物理的なリンクを1つの論理的なリンクとして束ねることができます。この機能は通常、「リンクアグリゲーション」または「ポートチャネル」などと呼ばれます。

LLDP

  • LLDP（Link Layer Discovery Protocol）は、ネットワークデバイス間でのリンク層（データリンク層）の情報を交換するためのプロトコルです。IEEE 802.1AB標準に基づいています。LLDPは、ネットワークデバイスが物理的に接続されているリンク上の情報を相互に通知するために使用され、ネットワークトポロジの自動検出や設定の簡素化に貢献します。

Longest Match

  • 最長一致ルール（Longest Match）。ルーティングテーブルを評価する際に、最長一致ルール（Longest Match）を適用します。これは、送信先のIPアドレスに対するルーティング情報のエントリーの中で、もっとも長く一致するプレフィックス（サブネットマスクによって定義される）を優先して選択する方式です。
  • 記憶によると、RHELはテーブルを上から評価していったはずなので違いに注意しておくこと

MAC フラッド攻撃

  • ネットワークスイッチに対して偽の MAC アドレスを大量に送信することで、スイッチのリソースに過負荷を与える攻撃です。

MMS

  • Manufacturing Message Specification（MMS）プロトコルは、IEC 61850 プロトコルの 1 つです。MMS を使用すると、さまざまな製造およびプロセス制御のデバイス間で遠隔監視制御・情報取得（SCADA）データをリアルタイムで転送できます。MMS プロトコルは、TCP ポート 102 を使用して、クライアントデバイスとサーバーデバイス間でメッセージを交換します。

NMS

  • SNMP（Simple Network Management Protocol）の文脈でいう「NMS」は、「Network Management System」の略で、ネットワークを監視し、管理するためのシステムを指します。NMSは、SNMPを使用してネットワークデバイスやサーバーなどの機器から情報を取得し、ネットワーク全体の状態を監視し、適切な管理を行います。

OSPF

  • OSPF（Open Shortest Path First）は、リンクステート型のルーティングプロトコルであり、内部ゲートウェイプロトコル（IGP）の1つです。OSPFは、複数のルーター間でネットワークトポロジー情報を交換し、最適な経路を計算してルーティングテーブルを構築します。これにより、データパケットがネットワーク内を最短経路で転送されます。

Dead 値

  • Hello パケットを受信できない状態になってからOSPFネイバーがダウンしたと判断するまでの時間
  • 常に Hello 値の 4 倍

Hello 値

  • Hello パケットの送信間隔

Hello パケット

  • OSPF の処理を行うときに最初に送受信するパケット。これでネイバーの発見や維持を行う。

OSPFプロセスID

  • OSPF ルーティングプロトコルのインスタンスを識別するために使用される番号です。OSPFは、複数のエリアに分けることができ、各エリアには独立したOSPFプロセスIDが必要です。

OSPFエリア

  • OSPFネットワークを分割するための単位

PVST/PVST+

  • PVST（Per-VLAN Spanning Tree）とは、VLANごとに異なるスパニングツリーを形成する規格。Cisco プロプラなスパニングツリープロトコル。
  • 「＋」がつくとIEEEの標準規格。

SAE

  • SAE（Simultaneous Authentication of Equals）は、ワイヤレスネットワークのセキュリティに使用される暗号化手法の一つで、WPA3（Wi-Fi Protected Access 3）の一部として採用されています。

SFP

  • SFP（Small Form-factor Pluggable）は、ネットワーク機器で使用される小型の光ファイバーモジュールまたは銅ケーブルモジュール。ネットワークデバイスに容易に差し込み可能で、異なる接続タイプや伝送速度をサポートする。

VACL

  • VACL（VLAN Access Control List）は、VLANごとにトラフィックをフィルタリングするためのアクセス制御リスト。

VLAN ホッピング攻撃

  • VLANホッピング攻撃は、ネットワークセキュリティの脆弱性を悪用して、VLAN（Virtual Local Area Network）の境界を越えて通信する攻撃手法です。通常、VLANはセグメンテーションやセキュリティのために使用され、異なるVLANのデバイス同士は直接通信できません。しかし、VLANホッピング攻撃では、攻撃者はこの制約を乗り越えて異なるVLANにアクセスしようとします。

    • 想定シナリオ
      1. Double-Tagging（ダブルタギング）
        • VLAN タグを二重につけて異なるVLANにアクセスする攻撃。ネイティブVLANがタグをつけないことを悪用している。
      2. Native VLAN Exploitation（ネイティブVLANの悪用）
      3. Dynamic Trunking Protocol (DTP)の悪用
    • 想定対策
      1. 未使用のポートの無効化
      2. VLAN Pruning(VLANプルーニング)
      3. VLANのセキュリティ機能の有効化
        1. 自動トランキング防止
        2. トランク機能の無効など

VLSM

  • VLSM（Variable Length Subnet Mask）は、サブネットマスクの長さを変えることで、ネットワークアドレス空間を効率的に使用する技術です。

VTPサーバ

  • VTPサーバ（VLAN Trunking Protocol Server）は、VLAN情報を管理し、他のVTPクライアントおよびVTPリクエスターにVLAN情報を配布する役割を持つスイッチです。VTPサーバは、VLANの作成、変更、削除などの操作を行い、それらの情報をVTPドメイン内の他のスイッチに自動的に配信します。つまり、VTPサーバはVLANの中央管理を提供し、ネットワーク全体で一貫したVLAN構成を確保します。
    • トランスペアレントモードは、VTP（VLAN Trunking Protocol）で使用されるモードの1つであり、スイッチがVTPメッセージを受信しても、VLAN情報を更新せずに他のスイッチに転送するモードです。

WRED

  • WRED（Weighted Random Early Detection）は、ネットワークコネクションにおける輻輳管理の一形態であり、パケットドロップの制御に使用される手法です。WREDはトラフィックのキューにおいて、輻輳が発生する前に一部のパケットをドロップし、トラフィックを制御することを目的としています。このドロップは、トラフィックの流れを調整してネットワークの過負荷を回避するために行われます。

動的ARP検査

  • 動的ARP検査は、ARP（Address Resolution Protocol）トラフィックを監視し、不正なARP応答に対抗するセキュリティ機能。

ディスタンスベクタ

  • ディスタンスベクタは、ディスタンス(距離)とベクタ(方向)により最適なルートを決定するアルゴリズム。それをつかったプロトコルやね。
    • RIP v1 , v2
    • IGRP , EIGRP

トラフィックシェイピング

  • ネットワークトラフィックの送信速度を制御して、一定のレートやパターンでデータを送信する手法です。これにより、ネットワークの帯域幅を有効に管理し、トラフィックの均一化や優先順位の付与などが可能になります。
  • いわゆるシェイピングよ

中間者スプーフィング

  • ネットワークセキュリティの攻撃手法の一つです。攻撃者は、通信を行う2つの通信者の間に侵入し、通信を傍受したり、改ざんしたりすることが目的です。攻撃者は通信者の一方になりすまし、もう一方の通信者になりすまして、通信の中間に立って情報を盗み取ることが可能です。
    • ARP スプーフィング
    • DNS スプーフィング
    • SSL ストリッピング
      • 攻撃者はクライアントとサーバーの間に介在し、通信をSSL/TLS暗号化から非暗号化のHTTP通信にダウングレードします。

ネイティブVLAN

  • ネイティブVLAN（Native VLAN）は、トランクリンク上で送信されるタグのないフレームが所属するデフォルトのVLANです。通常、スイッチのトランクリンクでは、VLANに関する情報を持つ802.1Qタグが付加されたフレームが転送されますが、ネイティブVLANに属するフレームはこのタグがない状態で送受信されます。

プライマリリンク

  • ネットワークインターフェースの一種で、冗長構成において優先的に使用されるリンクを指します。

フローティングスタティックルート

  • フローティングスタティックルートとは、スタティックルーティングにおいて、デフォルトの経路以外に設定されるバックアップのルートを指します。通常のスタティックルートは、その経路が利用可能である場合にのみルーティングテーブルに採用されますが、フローティングスタティックルートは、通常のルートが利用不可能になった場合にのみ使用されます。
  • 具体的には、通常のスタティックルートには低いアドミニストラティブディスタンス（AD）が割り当てられ、優先度が高いため、通常の経路として選択されます。一方、フローティングスタティックルートにはより高いアドミニストラティブディスタンスが割り当てられます。したがって、通常のルートが利用不可能になった場合、より低いアドミニストラティブディスタンスを持つフローティングスタティックルートが自動的に選択されます。

ヘルパーアドレス(DHCP)

  • DHCP（Dynamic Host Configuration Protocol）をサポートするヘルパーアドレスは、ネットワークデバイス（通常はルーターまたは Layer 3 スイッチ）が、異なるサブネットに存在するDHCPサーバーへのDHCPリクエストを中継するための機能です。
  • ヘルパーアドレスは、DHCPクライアントがブロードキャストでDHCPリクエストを送信する場合に、それを異なるサブネット上のDHCPサーバーに転送する役割を果たします。通常、DHCPメッセージはブロードキャストで送信されるため、異なるサブネットに存在するDHCPサーバーには直接到達できません。ヘルパーアドレスはこの問題を解決します。

ルーツガード

  • “ルーツガード”は、通常、スパニングツリープロトコル（STP）およびその派生であるRapid Spanning Tree Protocol（RSTP）やMultiple Spanning Tree Protocol（MSTP）において使用される概念です。ルーツガードは、ブリッジングプロトコルにおいてルートブリッジ（またはルートスイッチ）として選出されたスイッチが再度ネットワークに参加した場合における安定性の向上を目的としています。

ルートフラッピング

  • ルートフラッピングは、ネットワークでルーティング情報が短時間で何度も変動する現象を指します。これは通常、ネットワーク内で発生するルーティングの変更が頻繁すぎる場合に起こります。ルートフラッピングが発生すると、ネットワークの安定性やパフォーマンスに影響を与える可能性があります。

ルートブリッジ

  • ルートブリッジは、スイッチングネットワークにおいて、スイッチ間でのSTP（Spanning Tree Protocol）によるループを防止するために使用される概念です。STPは、ネットワーク内の冗長なリンクをブロックし、単一のパスを確立することでループを防ぐプロトコルです。ルートブリッジは、STPにおいてネットワーク上の全てのスイッチのルートパス（最短経路）を決定する役割を果たします。端的に言えば、ルートブリッジはネットワーク上の全てのスイッチ間の最短経路を決定し、ループを回避するためのパスを確立します。

ルートブリッジの選出方法

  1. Bridge IDの比較
    • ルートブリッジは、Bridge ID（ブリッジID）が最も小さいスイッチとなります。Bridge IDは、プライオリティ値（Priority）とMACアドレスで構成されます。デフォルトでは、スイッチのプライオリティ値は32768ですが、この値は変更可能です。
  2. プライオリティ値の比較
    • Bridge IDが同じ場合、プライオリティ値が最も小さいスイッチがルートブリッジになります。プライオリティ値が同じ場合、MACアドレスの比較が行われます。
  3. MACアドレスの比較
  • Bridge IDとプライオリティ値が同じ場合、MACアドレスが最も小さいスイッチがルートブリッジになります。
